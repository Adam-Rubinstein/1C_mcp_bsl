# Запуск JAR из командной строки

Сервер — обычное Java-приложение. Полезные опции (по справке `--help`):

| Опция | Короткая форма | Смысл |
|--------|------------------|--------|
| `--platform-path` | `-p` | Каталог установки **конкретной версии** платформы 1С (корень с `bin`, `docs`). |
| `--help` | `-h` | Справка по опциям. |
| `--verbose` | `-v` | Подробный лог в консоль. |
| `--mode` | `-m` | Режим: `stdio` (по умолчанию, для Cursor) или `sse` (HTTP). |
| `--port` | — | Порт для режима `sse` (по умолчанию 8080). |

Примеры:

```text
# Только проверка (выход сразу после справки)
java -Dfile.encoding=UTF-8 -jar путь\к.jar --help

# Режим stdio — так запускает Cursor; вручную процесс «висит» и ждёт stdin
java -Dfile.encoding=UTF-8 -jar путь\к.jar --platform-path "путь\к\1cv8\8.3.xx.xxxx"
```

Для **Cursor** используйте режим **stdio** (значение по умолчанию), в `mcp.json` **не** передавайте `--mode sse`, если не настраиваете отдельный HTTP-клиент.

## Лог в файл (опционально)

В `mcp.json` можно добавить блок `env`:

```json
"env": {
  "LOG_FILE": "C:/Tools/1C_mcp_bsl/mcp-server.log"
}
```

Путь к логу — без кириллицы, каталог должен существовать или быть создаваемым процессом.

## Откуда «код» инструментов `search`, `info`, …

Он **скомпилирован внутри JAR**. В этом репозитории лежит **готовый** файл `dist/1C_mcp_bsl.jar` и **инструкции** для Cursor; исходный **текст** сервера (до сборки) — в **`src/`** и Gradle-файлах в корне репозитория.

## Сборка JAR самостоятельно (опционально)

Если у вас есть **дерево исходников** того же сервера (Gradle-проект), типичная сборка:

```bash
./gradlew build
```

Готовый JAR обычно появляется в `build/libs/`. Детали зависят от вашей копии проекта; в данном репозитории сценарий сборки **не** описан.
